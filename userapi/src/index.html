Welcome to Tristan's and Apolline's userapi

Complete API documentation has been created using Swagger

Here's a table reviewing all available API methods and the path assigned to these methods:


CRUD Operations

Method Path Parameter Usage
POST /user none use this method on given path to add a new user to the REDIS DB. In the post request send a JSON body containing following information: {"username" = "", "firstname" = "", "lastname"=""}
PUT /user/updates none update the information of an existing user. Send the following json body: {"username" = "", "firstname" = "", "lastname"=""}
GET /user/:username username use this method on given path to retrieve a specific user (giving his username in URL) from the REDIS DB.
DELETE /user/:username username use this method on given path to delete a specific user (giving his username in URL ) from the REDIS DB.
GET /user/keys none use this method on given path to retrieve all keys present in the REDIS DB.

Other Routes


Method Path Parameter Usage
GET /api-docs none obtain complete documentation of the API
GET /health none obtain the health of the application

IMPORTANT: This application requires a REDIS DB to work perfectly. If you are using this application on AZURE or in a single Docker container not linked with a REDIS container, API methods won't work for you.


LINKS: except for the POST: /user/ endpoint other API endpoints require at least one database entry to be used.
